# 8.1 if标签

<if test="">

</if>
1
2
3
  • if标签中test属性是必须的
  • if标签中test属性的值是false或者true
  • test属性可以使用的是：
    • 当使用了@Param注解，那么test中要出现的是@Param注解指定的参数名，@Param("brand")，那么这里只能使用brand
    • 当没有使用@Param注解，test属性中要出现的是：param1、param2、...arg0、arg1、...
    • 当使用了POJO类，test属性中使用的是POJO的属性名
    • 在MyBatis的动态SQL中，不能使用&&，只能使用 and
    /**
     * 多条件查询

# 8.3 trim标签

  • prefix:加前缀
  • suffix:加后缀
  • prefixOverrides:删除前缀
  • suffixOverrides:删除后缀
        <!--
            prefix:加前缀
            suffix:加后缀
            prefixOverrides:删除前缀
            suffixOverrides:删除后缀
            -->
        <trim prefixOverrides="" suffix="" prefix="where" suffixOverrides="and|or">

# 8.4 set标签

  • 主要使用在update语句中，用来生成set关键字，同时去掉最后多余的 “ ，”
  • 比如我们只更新提交的不为空的字段，如果提交的数据是空或者“ ”，那么这个字段我们将不会更新
    /**
     * set 标签
     */
    int updateBySet(Car car);
1
2
3
4
    <update id="updateBySet">
        update t_car
        <set>
            <if test="carNum != null and carNum != ''"> car_num = #{carNum}, </if>
            <if test="brand != null and brand != ''"> brand = #{brand}, </if>
            <if test="guidePrice != null and guidePrice != ''"> guide_price = #{guidePrice}, </if>
            <if test="produceTime != null and produceTime != ''"> produce_time = #{produceTime}, </if>
            <if test="carType != null and carType != ''"> car_type = #{carType} </if>
        </set>
        where id = #{id}
    </update>

# 8.6 forEach标签

  • collection：遍历的数组或集合
  • item：数组或集合中集体的某一项
  • separator：遍历出来后每一项之间的分隔符
