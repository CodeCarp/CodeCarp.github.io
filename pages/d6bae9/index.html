# 2. 实现一个最基本的web应用

# 2.1 步骤

  • 第一步：找到CATALINA_HOME\webapps目录；

                 因为所有的webapps要放到webapps目录下（Tomcat服务器在这个目录下找搭建的应用）。
    1

  • 第二步：在CATALINA_HOME\webapps目录下新建一个子目录，起名 ：oa；

                  这个目录名oa就是这个webapp的名字。
    1

  • 第三步：在oa目录下新建资源文件。例如：index.html；

                  编写index.html文件的内容；
    1

  • 第五步：启动Tomcat服务器；8080代表的是Tomcat

                  <http://127.0.0.1:8080/oa/index.html>
    1

    在浏览器上输入一个URL和超链接一样；

    <!-- 注意一下的路径以 / 开始，是一个绝对路径，不需要添加：http://127.0.0.1:8080-->
<a href="oa/login.html">user login</a>

<!-- 多个层级也没有关系，正常访问！！-->
<a href="/oa/test/debug/d.html">d page</a>

# 2.2 B/S结构系统的角色和协议

# 2.3 有哪些角色（在整个BS结构的系统中，有哪些人参与进去了？）

  • 浏览器软件的开发团队（谷歌浏览器、火狐浏览器、IE浏览器）；
  • WEB Server的开发团队（Tomcat、Jetty、WebLogic、JBOSS、…）；
  • DB Server的开发团队（Oracle、MySQL、…）；
  • webapp的开发团队

# 2.4 角色和角色之间需要遵守哪些规范？哪些协议？

  • webapp的开发团队 和 WEB Server的开发团队 之间有一套规范：JavaEE规范之一Servlet规范；

       servlet规范的作用是什么？

  • Browser 和 WebServer之间有一套传输协议：HTTP协议。（超文本传输协议）

  • webapp开发团队 和 DB Server的开发团队之间有一套规范：JDBC规范。
