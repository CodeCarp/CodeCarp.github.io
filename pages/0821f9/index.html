# 1、Spring启示录

# 1.1 程序的缺点

  • 违背OCP开闭原则
  • 违背DIP依赖倒置原则

# 1.2 控制反转IOC

  • 控制反转：IOC
  • 反转是什么？
    • 不在程序中采用硬编码的方式new对象
    • 不在程序中采用硬编码方式来维护对象的关系
    • 将new对象的权力和对象之间的维护权全部交出去了
  • 控制反转是一种编程思想，或者叫做一种新的设计模式，由于出现的比较新，没有纳入GOF23种设计模式范围

# 1.3 spring框架实现IOC

  • Spring框架实现了控制反转IOC这种思想
    • Spring可以帮你new对象
    • Spring可以帮你维护对象之间的关系
    • Spring是一个实现了IOC思想的容器
  • 控制反转的实现方式有多种：其中比较重要的是：依赖注入（Dependency Injection，简称DI）
  • 控制反转是思想，依赖注入是这种思想的具体实现
  • 依赖注入DI，包括常见的两种方式;
    • 第一种方式：set注入（执行set方法给属性赋值）
    • 第二种方式：构造器方法注入（执行构造方法给属性赋值）
  • 依赖注入中“依赖”是什么意思，“注入”是什么意思？
    • 依赖：A对象和B对象的关系
    • 注入：是一种手段，通过这种手段，可以让A对象和B对象产生关系
