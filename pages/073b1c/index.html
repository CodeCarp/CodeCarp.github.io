# 23. Filter过滤器

# 23.1 Filter过滤器是什么？

  • Filter过滤器可以在Servlet这个目标程序执行之前添加代码，也可以在目标程序执行之后添加代码（过滤规则）
  • 一般情况下载过滤器中编写公共代码

# 23.2 Filter的实现

  • 第一步：编写一个Java类实现接口：jakarta.servlet.Filter，并且实现这个接口中的所有方法

    public class MyFilter implements Filter {
    @Override
    public void init(FilterConfig filterConfig) throws ServletException {
        // 在Filter对象第一次被创建之后调用，并且只调用一次
    }

    @Override
    public void doFilter(ServletRequest servletRequest, ServletResponse servletResponse, FilterChain filterChain) throws IOException, ServletException {
		// 只要用户发送一次请求，则执行一次，发送N次请求，则执行N次，在这个方法中编写过滤规则
    }

    @Override
    public void destroy() {
		// 在Filter对象被释放/销毁之前调用，值调用一次
    }
}

    3. Servlet是单例的，Filter也是单例的

# 23.3 目标程序是否执行，取决于两个条件

  • 第一：在过滤器当中是否编写了：chain.doFilter(request,response);代码

  • 第二：用户发送的请求路径是否和Servlet的请求路径一致

    chain.doFilter(request,response) // 作用：执行下一个过滤器，如果没有过滤器了，执行最终的Servlet
    1

  • 注意：Filter的优先级，天生的就比Servlet优先级高

    • /a.do，对应一个Filter，同时也对应一个Servlet的话，那么一定是先执行Filter，然后执行Servlet

# 23.4 Filter的执行顺序

  • 在web.xml文件中进行配置的时候，Filter的执行顺序是：
    • 依靠filter-mapping标签的配置位置，越靠上优先级越高
  • 过滤器的调用顺序，遵循站数据结构
  • 使用@WebFilter的时候，Filter的执行顺序：
    • 执行顺序是：比较Filter这个类名
    • 比如：FilterA和FilterB，则先执行FilterA
    • 比如：Filter1和Filter2，则先执行Filter1

# 23.5 Filter过滤器的生命周期

  • 和Servlet对象生命周期一致
  • 唯一的区别就是：Filter默认情况下，在服务器启动阶段就实例化，Servlet不会。
