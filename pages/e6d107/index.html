# 6、SpringMVC拦截器

# 6.1 拦截器概述

  • 过滤器Filter在任何基于servlet的框架都可以使用，而拦截器Interceptor是SpringMVC独有的
  • 过滤器Filter配置/*拦截所有资源，所有静态资源都会被拦截，而拦截器Interceptor只会拦截控制器Controller的方法

# 6.2 自定义拦截器

定义拦截器必须实现HandlerInterceptor接口，并重写接口中的3个方法，

  • preHandler方法：该方法在控制器的处理请求方法前执行，其返回值表示是否中断后续操作，返回true表示继续向下执行，返回false表示中断后续操作
  • postHandler方法：该方法在控制器的处理请求方法调用后，解析视图之前执行，可以通过此方法对请求域中的模型和视图做进一步的修改
  • after方法：该方法在控制器的处理请求方法执行完成之后，即视图渲染结束后执行，可以通过此方法实现一些资源清理，记录日志信息等工作
/**
 * 定义拦截器
 */
public class TestInterceptor implements HandlerInterceptor {
    //主要逻辑：在Controller之前执行：抽取Controller中的冗余代码
    @Override
    public boolean preHandle(HttpServletRequest request,
                             HttpServletResponse response, Object handler) throws Exception {
        System.out.println("pre...");
        return true;//放行，后续的拦截器或Controller就会执行
    }
    //在Controller之后执行:进一步的响应定制
    @Override
    public void postHandle(HttpServletRequest request, HttpServletResponse response, Object handler, ModelAndView modelAndView) throws Exception {
        System.out.println("post...");
    }
    //在页面渲染完毕之后，执行：资源回收
    @Override
    public void afterCompletion(HttpServletRequest request,
                                HttpServletResponse response, Object handler, Exception ex)
            throws Exception {
        System.out.println("after...");
    }
}

# 6.3 配置拦截器

<!-- 配置拦截器 -->
<mvc:interceptors>
    <mvc:interceptor>
        <!-- 拦截器拦截的路径 /test1下的任意多级路径 -->
        <mvc:mapping path="/test1/**"/>
        <!-- 拦截器拦截的路径 特定路径 -->
        <mvc:mapping path="/test2/test1"/>
        <!-- 不拦截此路径 -->
        <mvc:exclude-mapping path="/test1/test2"/>
        <bean class="tech.code2048.interceptor.TestInterceptor" />
    </mvc:interceptor>
</mvc:interceptors>
