# 16. Servlet中的注解式开发

# 16.1 注解式开发的优点

  • Servlet3.0版本之后，推出了各种Servlet基于注解式的开发
  • 注解式开发效率高，不需要编写大量的配置信息，直接在java类上使用注解进行标注
  • web.xml文件变小了
  • 并不是有了注解之后，web.xml文件就不需要了
  • 有一些需要变化的东西还是需要配置到wen.xml文件中，一般都是 注解+配置文件 的开发模式
  • 一些不会经常变化修改的配置建议使用注解，一些可能会被修改的信息建议写到配置文件中

# 16.2 第一个注解

  • javax.servlet.annotation.WebServlet;
    1

  • 在Servlet类上使用：@WebServlet，WebServlet注解中有哪些属性？

    • name属性：用来指定Servlet的名字，等同于：<servlet-name></servlet-name>
    • urlPatterns属性：用来Servlet的映射路径，可以指定多个字符串，等同于：<url-pattern></url-patern>
    • loadOnStartUp属性：用来指定在服务器起送阶段是否加载该Servlet。等同于：<load-on-startup></load-on-startup>
    • value属性：当注解的属性名是value时，使用注解的时候，value属性名可以省略
    • 注意：不是必须将所有属性都写上，只需要提供需要的

# 16.3 注解对象的使用格式

  • @注解名称（属性名=属性值，属性名=属性值，属性名=属性值，属性名=属性值......）

  • @WebServlet(name = "HelloSer",urlPatterns = {"/hello","/hello1"},loadOnStartup = 1,
        initParams = {@WebInitParam(name = "user",value = "root"),@WebInitParam(name = "password",value ="123")})
public class HelloServlet extends HttpServlet {
    @Override
    protected void doGet(HttpServletRequest request, HttpServletResponse response)
            throws ServletException, IOException {
        response.setContentType("text/html;charset=utf-8");
        PrintWriter out = response.getWriter();

        ServletConfig config = this.getServletConfig();
        String servletName = config.getServletName();
        out.print("servletName = " + servletName + "<br>");

        `Enumeration<String>` names = config.getInitParameterNames();
        while (names.hasMoreElements()) {
            String name = names.nextElement();
            String parameter = getInitParameter(name);
            out.print("name = " + name + ",password = " + parameter + "<br>");
        }

        String path = request.getServletPath();
        out.print("path = " + path);


    }
}
