  • 为什么要忽略他们？

    • 与项目的实际功能无关，不参与服务器上部署运行，把他们忽略能够屏蔽IDE工具之间的差异

    • 创建忽略文件 xxx.ignore （前缀名随便起，建议是 git.ignore）

# 7.3 初始化本地库

  • 选择要创建Git本地仓库的工程

# 7.4 添加到暂存区

  • 点击项目选择Git ->Add将项目添加到缓存区

# 7.5 提交到本地库

# 7.6 切换版本

  • 在IDEA中左下角，点击 Version Control，然后点击Log查看版本

  • 右键选择要切换的版本，然后在菜单里点击 checkout Revision

# 7.7 创建分支

  • 选择Git，在Repository里面，点击Branches按钮

  • 在弹出的Git Brances框里面，点击 New Branch按钮

  • 填写分支名称，创建hot-fix分支

  • 然后再IDEA的右下角看到hot-fix，说明分支创建成功，并且当前已经切换成hot-fix分支

# 7.8 切换分支

  • 在IDEA窗口右下角，切换到master分支

  • 然后再IDEA窗口的右下角看到master，说明master分支切换成功

# 7.9 合并分支

  • 在IDEA窗口的右下角，将hot-fix分支合并到当前master分支

  • 如果代码没有冲突，分支直接合并成功以后，代码自动提交，无需手动提交到本地库

# 7.10 解决冲突

  • 如果在合并分支的时候出现冲突

  • 点击Conflicts框里面的Merge按钮，进行手动合并代码

  • 手动合并完代码，点击右下角的Apply按钮

  • 代码冲突解决，自动提交本地库
