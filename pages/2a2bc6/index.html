# 21. EL表达式

# 21.1 EL表达式是什么

  • Expression Language（表达式语言）
  • 可以代替JSP中的Java代码
  • EL表达式库算是JSP语法的一部分，EL表达式归属于JSP

# 21.2 EL表达式的作用

  1. 从某个域中取数据
  2. 将取出的数据转成字符串
  3. 将字符串输出到浏览器

# 21.3 EL表达式的语法格式

  • ${表达式}

  • <%
	// 创建User对象
	User use = new User();
	user.setUsernam("jackson");
	user.setPassword("123");
	user.setAge("18");

	// 将User对象存储到某个域中，一定要存，因为EL表达式只能从域中取数据
	// 数据时必须存储到四大范围之一的
	request.setAttribute("user",user);
%>
<%--使用EL表达式取--%>
${user}  // 等同于Java代码：<%=request.getAttrbute("user")%>

面试题：
	${abc}和${"abc"}的区别是什么？
		${abc}表示从某个域中取出数据，并且被取的数据的name是“abc”，之前一定有这样的代码：域.serAttrbute("abc",对象)；
		${"abc"}表示直接将“abc"当做普通字符串输出到浏览器，不会从某个域中取数据

${user} 底层是怎么做的？从域中取数据，取出user对象，然后 调用user对象的toString方法，转换成字符串，输出到浏览器

<%--如果想输出对象的属性值--%>、
${user.username} Username必须有getUsername()方法，底层调用了getUsername()方法
${user.password} Username必须有getpassword()方法，底层调用了getpassword()方法

EL表达式中“.”这个语法，实际上是调用了底层的getxxx()方法
注意：如果没有对应的get方法，则出现异常，报500错误

  • EL表达式优先从小范围中读取数据

    • pageContext < request < session < application

  • EL表达式中有四个隐含的范围：

    • pageScope 对应的是 pageContext范围
    • requestScope 对应的是 request范围
    • sessionScope 对应的是 session范围
    • applicationScope 对应的是application范围

  • EL表达式对null进行了预处理，如果是null，则向浏览器输出一个空字符串

  • <%page contentType="text/html;charset=UTF-8" %>

<%
	request.setAttribute("username","zhangsan");
%>

<%--取出数据并且输出到浏览器--%>
<%=request.getAttribute("username") %>
采用EL表达式：${username}

<br>

<%=request.getAttribute("user") %>
<%--EL主要任务是做页面展示，要求页面展示是友好的--%>
<%--所以EL表达式对null进行了预处理，如果是null，则在浏览器上显示空白--%>
采用EL表达式：${user}

<hr>
<%--EL表达式表面上是这种写法，实际上运行的时候，还是要翻译成java代码的--%>
${username} 这个EL表达式等同于这一段java代码：<%=request.getAttribute("username") == null ? "" : request.getAttribute("username")%>

  • EL表达式取数据的时候有两种形式：

    • 第一种： . （大部分使用这种形式）
    • 第二种： [ ] （如果存储到域的时候，这个name中含有特殊字符，可以使用[ ]）
      • request.setAttribute("abc.def","zhangsan");
      • ${requestScope.abc.def}这样是无法取值的
      • 应该是这样：${requestScope["abc.def"]}

  • 掌握使用EL表达式，怎么从Map集合中取数据

    • ${map.key}

  • 掌握使用EL表达式，怎么 从数组和List集合中取数据

    • ${数组[下标]} （越界将不会取到值，显示空白）
    • ${list[下标]}

  • page指令中，有一个属性，可以忽略EL表达式

  • <%@page contentType="text/html;charset=UTF-8" isELIgnored="true"%>
isELIgnored="true"表示忽略EL表达式
isELIgnored="false"表示不忽略EL表达式（默认情况）

isELIgnored="true"这个是全局忽略
可以使用反斜杠进行局部忽略：\${user}这样可以忽略EL表达式

# 21.4 JSP中EL表达式的隐含对象

  • pageContext、param、paramValues、initParam、其他

  • 对隐含对象的使用：

    • 获取应用得根路径：

      ${pageContext.request.contentPath}
      1

    • 获取请求的参数：localhost:8080/jsp?username=zhangsan

      用户名：<%=request。getParameter("username")%>
用户名：${param.username}
      1
      2

    • 获取复选框提交的参数：localhost:8080/jsp?aihao=somke&aihao=drink&aihao=tangtou

      爱好：<%=request.getParameterValues("aihao")%>  // 一维数组
爱好：${paramValues.aiaho[0]}、${paramValues.aiaho[1]}、${paramValues.aiaho[2]}
      1
      2

    • initParam，上下文参数

      每页显示条数：<%=application.getInitParameter("pageSize")%>
每页显示条数：${initParam.pageSize}

# 21.5 EL表达式中的运算符

  • 算术运算符（只做数字的运算，不做字符串的拼接）

    + - * / %

${10 + 20} <br>

<%-- 在EL表达式当中“+”运算符只能做求和运算。不会进行字符串拼接操作。 --%>
<%--"20"会被自动转换成数字20--%>
${10 + "20"} <br>

<%-- java.lang.NumberFormatException: For input string: "abc" --%>
<%-- + 两边不是数字的时候，一定会转成数字。转不成数字就报错：NumberFormatException--%>
\${10 + "abc"} <br>

\${"king" + "soft"} <br>

  • 关系运符（自行调用equals方法 ==和eq的效果相同）

    == != > >= < <= eq

<%-- 关系运算符 --%>
${"abc" == "abc"} <br>
${"abc"} == ${"abc"} <br>

<%
    Object obj = new Object();
    request.setAttribute("k1", obj);
    request.setAttribute("k2", obj);
%>
${k1 == k2} <br> <%--true--%>

<%
    String s1 = new String("hehe");
    String s2 = new String("hehe");
    request.setAttribute("a", s1);
    request.setAttribute("b", s2);
%>
${a == b} <%--true--%> <br>

<%
    Object o1 = new Object();
    Object o2 = new Object();
    request.setAttribute("o1", o1);
    request.setAttribute("o2", o2);
%>
${o1 == o2} <br> <%--false--%>

<%
    Student stu1 = new Student("110", "警察");
    Student stu2 = new Student("110", "警察");

    request.setAttribute("stu1", stu1);
    request.setAttribute("stu2", stu2);
%>

<%--EL表达式当中的 "==" 调用了equals方法。--%>
\${stu1 == stu2} <br>
\${stu1 eq stu2} <br>

<hr>
<%-- != 也会调用对象的equals方法。--%>
${stu1 != stu2} <br>
${200 != 200} <br>

<%--以下语法错误，没有加小括号--%>
\${!stu1 eq stu2} <br>
<%--正确的--%>
${!(stu1 eq stu2)}<br>
${not(stu1 eq stu2)}<br>
