# 10. HTTP协议

# 10.1 什么是协议？

  • 协议实际上是某些人、或某些组织提前制定好的一套规范，大家都按照这个规范来，这样做到沟通无障碍。
  • 协议就是一套规范，就是一套标准，由其他人或组织来负责制定的；
  • 我说的话你能听得懂，你说的话，我能听得懂，这说明我们之间的是有一套规范的，一套协议的，这套协议就是：中国普通话协议，我们都遵循这套协议，我们之间就可以沟通无障碍。

# 10.2 什么是HTTP协议？

  • Http协议是W3C制定的一种超文本传输协议。（通信协议：发送消息的模板提前被制定好。）

  • W3C：

    • 万维网联盟组织
    • 负责制定标准：HTTP、HTML、H5、XML、DOM等规范都是W3C制定的
    • 万维网之父：蒂姆·伯纳斯·李

  • 什么是超文本？

    • 超文本就是：不是普通的文本，比如流媒体：声音、视频、图片等。
    • HTTP协议支持：不但可以传送普通字符串，同样支持传递声音、视频、图片等流媒体信息

  • 这种协议游走在B和S之间，B向S发送数据要遵循HTTP协议，S向B发送数据同样需要遵循HTTP协议，这样B和S才能解耦合。

  • 什么是解耦合？

    • B不依赖S
    • S不依赖B

  • 浏览器 向 WEB服务器发送数据，叫做：请求（request）

  • WEB服务器 向 浏览器发送数据，叫做：响应（response）

  • HTTP协议包括：

    • 请求协议
      • 浏览器 向WEB服务器发送数据的时候，这个发送的数据需要遵循一套标准，这套标准中规定了发送的数据具体格式
    • 响应协议
      • WEB服务器 向 浏览器发送数据的时候，这个发送的数据需要遵循一套标准，这套标准中规定了发送的数据具体格式

  • HTTP协议就是提前制定好的一种消息模板。

    • 不管你是哪个品牌的浏览器，都是这么发
    • 不管你是哪个品牌的WEN浏览器，都是这么发
    • FF浏览器 可以向 Tomcat发送请求，也可以向Jetty服务器发送请求，浏览器不依赖具体的服务器品牌。
    • WEB服务器也不依赖具体的浏览器，可以是FF浏览器，也可以是Chrome浏览器，可以是IE，都行。

    • 请求行

      • 包括三部分：
        • 第一部分：请求方式（7种）
          • get（常用的）
          • post（常用的）
          • delete
          • put
          • head
          • options
          • trace
        • 第二部分：URL
          • 什么是URI？统一资源标识符。代表网络中某个资源的名字，但是通过URI无法定位资源
          • 什么是URL？统一资源定位符。代表网络中某个资源，同时，通过URL是可以定位到该资源的
          • URI和URL什么关系？有什么区别？
            • URL包括URI
            • http://localhost:8080/servlet/index.html这是URL
            • /servlet/index.html这是URI
        • 第三部分：Http协议版本号
        • 请求头：
          • 请求的主机
          • 主机的端口
          • 浏览器信息
          • 平台信息
          • coolie等信息
        • 空白行
          • 空白行使用来区分“请求头”会“请求体”的
        • 请求体
          • 向服务器发送的具体数据

  • 状态行
    • 三部分组成
      • 第一部分：协议版本号（HTTP/1.1）
      • 第二部分： 状态码
        • 200表示 请求响应成功，正常结束
        • 404 表示访问的资源不存在，通常是因为路径写错了，要么就是路径写对了。风扇呢hi服务器对应的资源并没有启动成功，404是前端错误
        • 405表示前端发送的请方式与后端请求的处理方式不一致时发生的
          • 比如：前端是POST请求，后端的处理方式按照get方式进行处理时，发生405
          • 比如：前端是GET请求，后端的处理方式post时，发生405
        • 500表示服务器端的程序发生错误，
        • 以4开头的，一般是浏览器的错误导致的
        • 以4开头的，一般都是服务器端的错误导致的
      • 第三部分：状态的描述信息
        • ok 表示正常
        • not found表示资源找不到
  • 响应头：
    • 响应的内容类型
    • 响应的内容长度
    • 响应的时间
    • …
  • 空白行
    • 用来分割“响应头”和“响应体”的
  • 响应体
    • 响应体就是响应的正文，这些内容是一个长的字符串，这个字符串被浏览器渲染，解释并执行，最终展示出效果

# 10.4 怎么向服务器发送GET请求？怎么向服务器发送POST请求？

  • 到目前为止，只有一种情况发送POST请求，使用form表单，并且form标签中的method属性值为：method=“post”
  • 其他情况一律用get请求：
    • 在浏览器地址栏上直接输入URL，属于get请求；
    • 在浏览器上直接点击超连接，属于get请求
    • 使用form表单提交数据时，form标签中没有写method属性，默认就是get
    • 或者使用form的时候，form标签中method属性值为：method=“get”
    • …
